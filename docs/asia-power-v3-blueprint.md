## 五、列表页 3.0 — 双模式设计

**核心原则：拆车件是主营业务；不同类目用不同展示模式。**

| **二手车 Used Vehicles** | **大图卡片 Grid** | 同上，整车更需照片评估 |

### 5.2 发动机型号页（如 2AZ-FE）— 两段式结构

**上方：发动机型号信息（SEO + 选型）**
- 排量、燃料、适配车型（Camry / RAV4 / Alphard…）
- 在库总台数、分拆价比例说明

**下方：在库实例列表（NOT 零配件）**

每一行 = **一台具体半车分拆出来的发动机**，对应一个底盘号：

```
┌──────────────────────────────────────────────────────────────┐
│ [发动机舱图 140×90]  JTDBE32K****1234 · ENG-HC250160        │
│                      Toyota Camry 2007 · 2AZ-FE              │
│                      68,000 km · Auto U151E · FWD · Video      │
│                                              EXW $1,853  →   │
└──────────────────────────────────────────────────────────────┘
```

- **照片**：发动机舱实拍（从半车照片集中自动选取或供应商上传）
- **底盘号**：脱敏 VIN / 车架号，便于买家核对
- **里程、年款、车型、变速箱、驱动**：来自半车主记录
- **库存 ID**：`ENG-HC250160`，链接 `← HC250160`
- **不展示**：活塞/缸盖等「组成发动机的零配件」

变速箱型号页（如 U151E）、底盘型号页结构相同，只是分拆比例为 35% / 20%。

### 5.3 发动机目录索引页 — 紧凑列表

`/engines/` 索引仍用紧凑列表展示**发动机型号**（2AZ-FE、K24A…），点击进入 5.2 型号页看具体在库实例。

## 六、库存分拆逻辑（半车 → 三套配件）

**规则：每上线一台半车，自动在发动机 / 变速箱 / 底盘目录各增加一套对应库存。**

### 6.1 价格分配比例

| 配件 | 占整车 EXW 价 | 示例（整车 $2,850） |
|------|---------------|---------------------|
| 发动机 Engine | **65%** | USD 1,853 |
| 变速箱 Gearbox | **35%** | USD 998 |
| 底盘 Chassis | **20%** | USD 570 |

> 分拆参考价总和为 120%，反映单买配件的溢价；正式报价以销售确认为准。

### 6.2 库存 ID 规则

```
半车主记录:  HC250160
发动机衍生:  ENG-HC250160  (或沿用 engine slug + sourceStockId 字段)
变速箱衍生:  GB-HC250160
底盘衍生:    CH-HC250160
```

### 6.3 数据模型（JSON 字段）

```javascript
{
  stockId: "HC250160",
  priceUsd: 2850,
  maskedVin: "JTDBE32K****1234",  // 底盘号（脱敏）
  derivedListings: [
    {
      type: "engine",
      stockId: "ENG-HC250160",
      engineCode: "2AZ-FE",
      priceUsd: 1853,
      ratio: 0.65,
      photo: "engine-bay-from-halfcut",  // 发动机舱图
      mileage: "68,000 km",
      model: "Camry",
      year: 2007,
    },
    // gearbox, chassis 同理
  ]
}
```

### 6.4 状态同步

- 半车整售 → 三套配件同步标记 Sold
- 任一套件售出 → 半车标记 Partial / 其余配件更新
- 半车改价 → 三套配件价格按比例重算
- **无变速箱时**：不生成 GB 记录，比例不重新分配（或 CEO 确认后调整）

### 6.5 触发时机

半车 Admin 批准上线时，`server/lib/inventory-cascade.js` 自动生成并写入：
- `data/half-cut-approved.json`（主记录）
- 发动机目录 JSON / powertrain memory
- 变速箱目录
- 底盘目录

## 八、搜索权重算法

```javascript
score = (
  freshnessWeight   * daysSinceUpdateScore +   // 越新越高
  demandWeight      * inquiryCountScore +      // 询价次数
  mediaWeight       * hasVideoPhotoScore +     // 有视频+5
  availabilityWeight* statusScore +            // Available > Reserved
  priceWeight       * hasPriceScore            // 有标价 +3
)
```

- 每次搜索记录 `{ keyword, timestamp, resultCount, userCountry }` → `data/search-keywords.json`
- Admin 面板：Top 20 关键词、零结果词（选品信号）
- 首页搜索栏占导航栏下方固定位置（Sticky）

## 十五、半切车 & 报废车 & 二手车说明

### 14.1 半切车（Half-Cut）定义

> 半切车指按买家要求切割的报废车辆残骸。**即使照片显示完整车身，默认已拆除：**
> - 三元催化器（TWC）
> - 铝合金轮圈（Alloy wheels）
>
> 买家可在确认订单前要求补充照片或视频验证。

### 14.2 报废车出口（中国法规）

> 根据中国法规，报废车辆出口须同时满足：
> 1. 在**中国车管系统正式注销**
> 2. 完成**切割处理**后方可出口
>
> 此要求适用于 Half-Cuts 类目，不适用于 Used Vehicles 类目。

### 14.3 二手车（Used Vehicles）类目

- 单独陈列，标签：**Export-Ready · Documentation Available**
- 备注字段：`exportType: whole-vehicle`, `docsAvailable: true`
- 详情页说明：可整车出口、手续齐全
- 与 Half-Cuts 互斥分类，不可混列

**展示位置：** 首页第 3 入口 + 独立 `/used-vehicles/` 目录

| **Sale Agreement** | 确认订单/付款 | B2B Sale of Goods Act 1979 |
| **Supplier Agreement** | 供应商入驻 | 独家/非独家、佣金、数据使用 |
| **Acceptable Use** | 平台通信 | 禁止绕过平台交易 |

### 15.2 中国大陆 vs 英国法 — CEO 需注意

| 议题 | 中国习惯 | 英国要求 |
|------|----------|----------|
| 合同 | 口头+微信常见 | **书面合同**为主，电子签名有效（Electronic Communications Act 2000） |
| 隐私 | 较宽松 | **UK GDPR**：明示同意、数据可删除权、跨境传输需 SCC |
| 消费者保护 | 三包法规 | B2B 豁免 Consumer Rights Act，但**必须在 Terms 声明 B2B only** |
| 退款 | 协商 | Sale Agreement 须明确 **Refund / Cancellation** 条款 |
| 争议解决 | 本地法院 | 建议 **England & Wales 管辖** + 仲裁选项 |
| 产品责任 | 产品质量法 | **Product Liability Act 1987** — 二手设备需明确「as-is」免责声明 |
| 出口合规 | 中国海关/车管 | 网站须声明**买家负责目的国进口合规** |

**建议：** 找英国商业律师审核 Terms + Sale Agreement（预算 £1,500–3,000 一次性）。
