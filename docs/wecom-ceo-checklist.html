企业微信 · AsiaPower 库存 Agent — CEO 一步一步

把销售 Agent 接到企业微信群：群里 @AsiaPower 库存 Agent 问销售数据，它会自动回复。我们内部叫它「子敬」。电脑已登录企业微信不够，还要进管理后台拿密钥。

企业微信里显示我们内部叫
AsiaPower 库存 Agent子敬
最快方式：在 Cursor 里对 AI 说「帮我填子敬配置」，把下面 5 个值发过去；或终端运行 .venv/bin/python3 scripts/wecom-setup-wizard.py
1

打开管理后台

浏览器打开：work.weixin.qq.com/wework_admin/frame

「管理后台」= 网页版控制台，不是桌面企业微信。看不到「应用管理」就找公司 IT 开管理员权限。

📷 截图占位：登录后左侧菜单「应用管理」
2

复制企业 ID

左侧 我的企业 → 企业信息 → 复制「企业 ID」

📷 截图占位：企业 ID 位置
3

创建「AsiaPower 库存 Agent」应用

应用管理 → 应用 → 创建应用，名称填 AsiaPower 库存 Agent（不要填「子敬」— 那是内部昵称），可见范围先选您和测试同事。

进入应用详情，复制 AgentId 和 Secret（Secret 要点「查看」）。

📷 截图占位：AgentId 与 Secret
4

配置接收消息（回调）

同一应用页 → 接收消息 → 设置 API 接收

「回调」= 企业微信把群消息推到我们的服务器。生产环境已部署，直接填下面三行即可（Token / Key 与服务器 .env 一致）。

  1. URL：https://asia-power.cn/wecom/callback（企业备案域名，与企微主体一致）
  2. Token：asiapower1
  3. EncodingAESKey：t5CiHUAFe76N9EryQq1USCskNLI5k1BbALUTdggTsHD
  4. 保存后 → 我的企业 → 企业信息 → 企业可信 IP → 添加 124.222.191.164

本地开发才用 ngrok；生产勿填 asia-power.com（个人备案，企微校验过不去）。

📷 截图占位：接收消息 API 设置页
5

启动本机服务 + ngrok

双击 docs/企业微信-子敬-一步一步.command（回调端口 8791）

另开一个终端：ngrok http 8791，把 https 地址填进上一步 URL 和 .env。

「ngrok」= 临时把本机端口暴露到公网，让企业微信能访问；意思是开发测试用的隧道工具。

6

把 Agent 拉进群并测试

企业微信 → 进测试群 → 添加应用 AsiaPower 库存 Agent → 群里发 @AsiaPower 库存 Agent /ping

成功：Agent 回复在线状态（回复里可能自称「子敬」）

📷 截图占位：群内 @AsiaPower 库存 Agent
7

验证配置

终端运行：.venv/bin/python3 scripts/wecom-verify-config.py

应显示 ✅ access_token 获取成功
